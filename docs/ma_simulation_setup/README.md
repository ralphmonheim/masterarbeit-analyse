# ma_simulation_setup

- **Zweck:** Run, Variantenmenge, Zeitraum, Zeitschritt, Ausgabeintervall und Szenario festlegen.
- **Eingaben:** ausgewaehlte Varianten sowie Projekt- und Wetterreferenzen.
- **Ausgaben:** validierte Run- und Simulationskonfiguration.
- **Abgrenzung:** keine Variantenbildung, keine Dateibearbeitung und kein Simulationsstart.
- **Abhaengigkeiten:** `ma_variants`; Phase 3.
- **Status:** geplant.
- **Naechster Schritt:** Run-Modell, ID-Regeln und Standard-Jahressimulation planen.
