# ma_feedback

- **Zweck:** Auffaelligkeiten klassifizieren und Rueckspruenge in verantwortliche Module steuern.
- **Eingaben:** Fehler, Warnungen sowie Analyse- und Bewertungsergebnisse.
- **Ausgaben:** Ruecksprungziel, Korrekturauftrag und dokumentierte Iteration.
- **Abgrenzung:** keine automatische Aenderung von Fachmoduldaten.
- **Abhaengigkeiten:** `ma_validation`, `ma_workflow`; phasenuebergreifend.
- **Status:** geplant.
- **Naechster Schritt:** Problemtypen, Prioritaeten und Ruecksprungregeln festlegen.
